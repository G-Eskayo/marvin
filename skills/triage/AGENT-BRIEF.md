# Writing Durable Agent Briefs

An agent brief is what turns a triaged issue into something an AFK agent (or a human, for
`ready-for-human`) can act on with zero further back-and-forth. The bar: a reader with no memory
of the triage conversation should be able to execute from the brief alone.

## Structure

```markdown
> *This was generated by AI during triage.*

## Agent Brief

**Goal**: [one sentence — what should be true when this is done]

**Context**: [the minimum a fresh agent needs — relevant files/modules, why this matters,
what's already been ruled out during triage and why, so it doesn't get re-litigated]

**Acceptance criteria**:
- [ ] Specific, checkable condition 1
- [ ] Specific, checkable condition 2

**Reproduction** (bugs only): [exact steps that reproduced the issue during triage, and the
confirmed code path — see triage SKILL.md step 3]

**Out of scope**: [anything explicitly NOT part of this — prevents scope creep during
implementation]

**Notes**: [anything a human would know instinctively but an agent needs stated — naming
conventions to follow, a related ADR to respect, a gotcha found during triage]
```

## What makes a brief durable vs. fragile

**Durable** — every claim is either verified during triage (a confirmed repro, a read file, a
quoted ADR) or explicitly marked as an assumption. A fresh agent can trust everything stated
without re-deriving it.

**Fragile** — assumes context from the triage conversation that isn't written down ("as we
discussed," "the usual approach here"), or states a plausible-sounding but unverified claim as
fact. If triage didn't confirm something, say "unverified: ..." rather than asserting it.

## For `ready-for-human` specifically

Same structure, plus a **Why not an agent** line explaining the specific reason — judgment call,
external system access an agent doesn't have, a design decision that needs a stakeholder, manual
testing that can't be automated. This isn't boilerplate — a vague reason here is a sign the issue
might actually belong in `ready-for-agent` instead.
